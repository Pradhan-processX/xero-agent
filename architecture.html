Xero Timesheet Agent - Current Architecture

Updated 2026-06-17. This page records the current custom-agent architecture used by the repo.

Stage: Phase 3.7 local bot integration testing Branch: feature/teams-bot Runtime: Node/Express + M365 Agents SDK LLM: Azure OpenAI via raw fetch
End-to-end flow

Microsoft Teams

User narrates work, reviews drafts, taps Submit or Cancel.

->

Azure Bot Service

Channel/auth gateway. Posts activities to /api/messages.

->

Node Backend

server.js, bot.js, agent.js, stores, Xero client.

->

Xero Projects API

Time entries are written only after user confirmation.

Bot shell

src/bot.js

  • M365 Agents SDK handler.
  • Loads conversation state.
  • Handles Submit/Cancel while in NEEDS_CONFIRMATION.
  • Routes all normal messages to agent.run().
Agent brain

src/agent.js

  • Triage: gpt-4.1-mini.
  • Reasoning: gpt-4.1.
  • Tools: get_projects, create_draft, get_week_summary.
  • Raw fetch(); no OpenAI SDK.
Persistence

Stores

  • conversationStore.js: per-chat state, 30-minute TTL.
  • draftStore.js: draft lifecycle and weekly summaries.
  • store.js: local JSON or Azure Table Storage.
Message handling sequence
Teams sends an activity to POST /api/messages.
server.js passes the turn to the M365 Agents SDK adapter.
bot.js resolves the Teams user through userMap.js.
bot.js loads conversation state from conversationStore.js.
If waiting for confirmation, Submit/Cancel is handled in app code with no LLM call.
Otherwise agent.run(text, history, user) handles the message.
Agent sequence
Triage model returns help, off_topic, review, or complex.
Help/off-topic return text immediately.
Review calls get_week_summary.
Complex messages enter the gpt-4.1 tool-calling loop.
get_projects fetches allowed projects/tasks for grounding.
create_draft validates and returns draft entries or clarification errors.
Guardrails in app code
Guard Where Behavior
Project allowlist agent.js create_draft Rejects projects outside the mapped user's allowed project IDs.
Task validity agent.js create_draft Task must belong to the selected project.
Duration required agent.js and prompt instruction Missing or invalid duration returns an error/clarification path.
Future date blocked agent.js create_draft Rejects dates after today.
No LLM Xero writes bot.js Submit handler Xero writes only happen after Adaptive Card Submit or typed submit in confirmation state.
